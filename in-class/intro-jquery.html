Introduction to Jquery

Getting Started

jQuery is a JavaScript Library. It is built from javascript. jQuery greatly simplifies JavaScript programming by reducing the amount of code you need to write. It also comes with methods that you can easily use to select and add behaviours to user actions when on a web page. jQuery is easy to learn. And works on many browsers! Neat!

There are two ways we can add jQuery to our websites. We can download it from jQuery.com. We can download the uncompressed version to develop with it or we download the minified version to load on to the server. We would use the script tag and point to our file using src attribute. Another way is to use a cdn aka content delivery network like google. Here is google's jquery cdn for current jQuery written at the time of that this page was written. Many users already have downloaded jQuery from Google or Microsoft when visiting another site. As a result, it will be loaded from cache when they visit your site, which leads to faster loading time. Also, most CDN's will make sure that once a user requests a file from it, it will be served from the server closest to them, which also leads to faster loading time.

Diving into the Syntax

With jQuery you select (query) HTML elements and perform "actions" on them.

Basic syntax is:

$(selector).action()

First we use the dollar symbol $ to define and access jQuery. Then we select/find the thing we want to perform an action on. When we select elements in css we can select elements in varying ways. We can select by element, by class, by id, by attributes, by first child, last child, nth child, by values within an attribute, etc.

The Document Ready Event

All of our jquery methods are written inside of a document ready event.

$(document).ready(function(){
    // jQuery methods go here...
});

We do this to prevent jquery from running code before the document is finished loading ie. is ready. It is good practice to wait for the document to be fully loaded and ready before working with it. Otherwise some methods can fail when these try to run and the document is not fully loaded.

You can also write the document ready event in shorthand. I don't recommend it as it is not as easy to read and or understand

$(function(){
   // jQuery methods go here...
});

You can write scripts within the head of your web page or you can write separate .js files that you link to your pages. This allows you to link the same script code to multiple pages within your site or only run certain scripts on some pages while others are free of scripts or have other scripts linked to it to perform other events.

Events

All the different visitor's actions that a web page can respond to are called events. An event represents the precise moment when something happens. Such events include when a user moves his or her mouse over an element, they select a button and or click on an element.We use the term fired up often with events.

DOM aka Document Object Model is a standard by which we access a document. With DOM HTML objects are treated as objects. The properties of all HTML elements can be defined via the DOM. We can use methods to access all HTML elements and we can also define actions to all html elements. In others the HTML DOM is a standard for how to get, change, add, or delete HTML elements.

Common DOM events include: click()/dblclick(), mouseenter()/mouseleave(), keydown()/keyup(), focus()/blur(), load(), resize(), etc.
